Stage 1: Work Session 3 Reflection

Cascading

Cascading allows for easier formatting by applying rules to elements, as well as their child elements, no matter where they are in the html document (inheritance). If an element has has more than one rule defined, the most specific rule for that element takes precendennce. Writing less CSS allows web pages to load more quickly and efficiently.

There are multiple ways to include CSS in a web page:

  • Write CSS in the head of your document
  • Link your HTML document to a separate CSS file
  • Write your style inline with your HTML

The benefits of CSS are many. In addition to reducing "bloat", the DRY (Dont Repeat Yourself) principle improves quality by making it easier for programmers to understand the code, make changes in as few places as possible, and allows for logic to be abstracted (business rules for example). The first step in eliminating uncessary repetition appears to be having the ability to recognize patterns.

Box Formatting

There are 4 parts to every box in HTML: Content, Padding, Border, Margin (from inside out). The size of each part can be set using a percentage or a pixel number. Different browsers can display the same information differently. There are techniques to avoid this issue.

Refining Your HTML and CSS

Developer Tools is an extreemly useful tool for refining your code. You can try many variations to instantly see the result, without actually making any changes to your code. You can also use websites such as the W3C Markup Validation Service to validate your code.

There is a step by step process you can use when designing an html document.

  • Look for natural boxes
  • Look for repeated styles and semantics
  • Write you your html
  • Apply styles (from biggest to smallest)
  • Fix things

Terminology

  • Program is what tells the computer what to do.
  • A Computer is a universal machine that runs programs.
  • Python is an programming language, and an interpreter.
  • An interpreter interprets the programming language into a language that the computer can understand.
  • Programming languages are needed because natural languages are to ambiguous (unclear) and verbose (wordy).

Grammar

Notating a language's grammar with "non-terminal -> replacement" is known as "Backus Naur Form." Backus Naur Form allows you to derive acceptable expressions for the language. You break down the expression until all your replacements are "non-terminals."

Python Expressions

In Python, the Grammar for arithmetic in Backus Naur Form looks like:

  • Expression -> Expression Operator Expression
  • Expression -> Number
  • Operator -> +
  • Operator -> *
  • Number -> 0, 1, ...
  • Expression -> (Expression)
(Bold items are non-terminals)

Stage 2: Work Session 2 Reflection

Variables

A Variable is an assignment statement, where a name is assigned an expression.

  • Example: Car = White Toyota
  • Print "Car" (Result: White Toyota)
The = sign for a Variable does not mean the same thing as in arithmetic, it is simply an assignment of a value. This value can change, and the Python interpreter will use the most recent value for each Variable. Variables are useful because they can improve the readability and simplify your code by replacing complex numbers or values with simply-named Assignment Statements.

Strings

A String is simply a set of characters, surrounded by quotes. Example: "I am a string!" You can use single quotes or double quotes, but whichever you choose, you have to be consistent on the front and end of a string. You can concatenate Strings, but you cannot add them. If you try to add two numbers with quotes around them, Python will just concatenate them, rather than add them together.

Indexing

Indexing allows you to extract portions of a String you are interested in. You can define the first and last portion of the string you want to extract (sub-sequence).

  • Example: Name = "Matt"
  • Print name[0:2]
  • Result: Ma

Find

Find is a Method (built-in procedure in Python). Find allows you to find the first position of the first occurrence of a target string, within a search string. If the function doesn't find the target string, Python will return a value of "-1". Otherwise, it will return the position number of just before the target string. A second parameter can be added to the find function to find the second occurrence of the target string.

Functions

Functions, or "procedures" take inputs, process them, and produce outputs. They consist of procedure(input,input,â€¦).

For example:
def sum(n1,n2):
return n1 + n2

This function would add two numbers together (and return a "3" in this example). In this example, "sum" is the function name, and "(n1,n2)" are the parameters. When calling (using) the function, you replace the parameters with values. These values could be numbers, strings, or even variables.

Making and Using Functions

Making a function is also called "defining." In order to use (call) a function, it must first be defined. Once defined, it can be used over and over in the program. This makes functions extremely useful tools, and simplifies code writing and debugging. If a function doesn't have a return statement, Python won't know what to do when it is called. By default, if Python can't understand what you want the function to do, it will return "None" when it is called.

Boolean Logic

Boolean logic compares variables and evaluates them as True or False. This logic is used in the construction of If, Or, and Else expressions in Python.

If and Else Expressions

If expressions consist of a test expression and block. They are written inside procedures. For example: def bigger(a,b):
if a > b:
    return a
return b

The above reads in english as follows: "If a is greater than b, return a. If not (or else), return b."

Else can be added and used used like this:
def bigger(a,b):
if a > b:
    return a
else: return 'the first number is smaller than the second number'

Else Expressions can also be used to simplify complex calculations. For example:
def biggest(a,b,c):
if a > b:
    if a > c:
         return a
    else:
         return c
else:
     if b > c
        return b
        return c

Stage 2: Work Session 5 Reflection

Structured Data

Lists are structured data because they contain defined positions. Strings are also structured, since their positions can also be queried.

Mutability

Lists are mutable, strings are not. Mutability means having the ability to change elements within the list, without creating a whole new list.

Append vs +

Appending a list to another list adds only one element to the initial list. This new element refers to the list that was appended. Therefore, changes to the appended list will be reflected in the list it was appended to. + works a lot like concatenate. If you concatenate two lists, each element in the appended list becomes a separate element in the list it was concatenated with.

For Loops

For Loops allow you to execute a piece of code multiple times (iterate), and are typically used when the number of iterations is known. Othwerwise, a While Loop could be used. For Loops iterate over a list or a string, in the order that they appear.
